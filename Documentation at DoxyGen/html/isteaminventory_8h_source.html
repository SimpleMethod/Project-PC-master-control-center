155  // RESULT SERIALIZATION AND AUTHENTICATION
156  //
157  // Serialized result sets contain a short signature which can't be forged
158  // or replayed across different game sessions. A result set can be serialized
159  // on the local client, transmitted to other players via your game networking,
160  // and deserialized by the remote players. This is a secure way of preventing
161  // hackers from lying about posessing rare/high-value items.
162 
163  // Serializes a result set with signature bytes to an output buffer. Pass
164  // NULL as an output buffer to get the required size via punOutBufferSize.
165  // The size of a serialized result depends on the number items which are being
166  // serialized. When securely transmitting items to other players, it is
167  // recommended to use "GetItemsByID" first to create a minimal result set.
168  // Results have a built-in timestamp which will be considered "expired" after
169  // an hour has elapsed. See DeserializeResult for expiration handling.
170  virtual bool SerializeResult( SteamInventoryResult_t resultHandle, OUT_BUFFER_COUNT(punOutBufferSize) void *pOutBuffer, uint32 *punOutBufferSize ) = 0;
171 
172  // Deserializes a result set and verifies the signature bytes. Returns false
173  // if bRequireFullOnlineVerify is set but Steam is running in Offline mode.
174  // Otherwise returns true and then delivers error codes via GetResultStatus.
175  //
176  // The bRESERVED_MUST_BE_FALSE flag is reserved for future use and should not
177  // be set to true by your game at this time.
178  //
179  // DeserializeResult has a potential soft-failure mode where the handle status
180  // is set to k_EResultExpired. GetResultItems() still succeeds in this mode.
181  // The "expired" result could indicate that the data may be out of date - not
182  // just due to timed expiration (one hour), but also because one of the items
183  // in the result set may have been traded or consumed since the result set was
184  // generated. You could compare the timestamp from GetResultTimestamp() to
185  // ISteamUtils::GetServerRealTime() to determine how old the data is. You could
186  // simply ignore the "expired" result code and continue as normal, or you
187  // could challenge the player with expired data to send an updated result set.
188  virtual bool DeserializeResult( SteamInventoryResult_t *pOutResultHandle, BUFFER_COUNT(punOutBufferSize) const void *pBuffer, uint32 unBufferSize, bool bRESERVED_MUST_BE_FALSE = false ) = 0;
